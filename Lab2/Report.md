# Результаты тестирования

## Параметры

Размер коллекций: 100000 элементов, 5 итераций, прогрев 3 раза.

## Основные результаты

List быстрее всего для добавления в конец и доступа по индексу. LinkedList лучше для вставки в начало и середину. Queue и Stack эффективны для своих операций. ImmutableList медленнее всех, но обеспечивает неизменяемость.

## Добавление в конец

List - 0.29мс, LinkedList - 2.34мс, ImmutableList - 29.45мс, Queue - 0.57мс, Stack - 0.37мс

## Добавление в начало

LinkedList - 2.08мс, ImmutableList - 30.62мс, List - 267.56мс

List очень медленный из-за сдвига всех элементов.

## Добавление в середину

LinkedList - 2.10мс, ImmutableList - 33.70мс, List - 117.46мс

## Удаление из конца

List - 0.25мс, Stack - 0.59мс, LinkedList - 3.30мс, ImmutableList - 23.90мс

## Удаление из начала

Queue - 0.77мс, ImmutableList - 13.51мс, LinkedList - 3.90мс, List - 269.42мс

List опять медленный из-за сдвига.

## Удаление из середины

ImmutableList - 13.63мс, List - 120.97мс, LinkedList - 8188.54мс

LinkedList здесь очень медленный, потому что нужно искать элемент каждый раз заново.

## Поиск

Queue - 0.07мс, List - 0.08мс, LinkedList - 3.96мс, Stack - 6.54мс, ImmutableList - 25.52мс

## Доступ по индексу

List - 0.17мс, ImmutableList - 4.46мс, LinkedList - 1.43мс

LinkedList медленный, так как нужно проходить по узлам.

## Выводы

List лучше всего для частого доступа по индексу и добавления в конец. LinkedList хорош для вставок в середину, но плох для доступа по индексу. Queue и Stack оптимальны для своих задач. ImmutableList медленный, но полезен для многопоточности.
